API Reference

IDPDR REST API v1.0 — developer documentation. Base URL: http://localhost:8000/api/. All endpoints require an authenticated session except /api/health/.

🧬
Patient-facing Endpoints
MethodEndpointDescription
GET/api/health/System health check (public, used to prime CSRF cookie)
GET/api/conditions/All 12 conditions with symptoms
POST/api/predict/4-model ensemble disease diagnosis
GET/api/recommend/?condition=XTop scored drugs for a condition
GET/api/recommend/all/Top drugs for all conditions
POST/api/sentiment/VADER NLP sentiment on drug review
GET/api/compare/?condition=X&drug_a=Y&drug_b=ZExtended drug comparison
GET/api/dashboard/Aggregated platform statistics
GET/api/drug/search/?q=XSearch drugs by name/generic/brand (empty q = browse all)
GET/api/drug/report/?drug=XFull drug profile
GET/api/drug/report/pdf/?drug=XDrug report as downloadable/printable PDF
GET/api/drug/buy/?drug=XAvailability, price, pharmacy references
POST/api/interactions/Multi-drug interaction severity checker
GET/api/disease/?condition=XDisease info: symptoms, treatment, emergency signs
POST/api/smart-recommend/AI personalized recommendation from symptoms
POST/api/diagnosis/report/Generate & persist a Disease Diagnosis Report
GET/api/diagnosis/reports/List the signed-in user's saved reports
GET/api/diagnosis/report/<id>/pdf/Diagnosis report as PDF
🛡
Admin-only Endpoints Require role = admin
MethodEndpointDescription
GET/api/admin/users/List/search all users
POST/api/admin/users/<id>/role/Promote/demote a user's role
POST/api/admin/users/<id>/toggle-active/Activate/deactivate a user
DELETE/api/admin/users/<id>/Delete a user account
GET / POST/api/admin/drugs/List or create managed drug entries
GET / PUT / DELETE/api/admin/drugs/<id>/View, update, or delete a managed drug
GET/api/admin/audit-logs/View/filter audit log events
GET / POST/api/admin/settings/View or update system settings
GET/api/admin/analytics/Extended platform analytics
